# Security policy

Please report vulnerabilities privately to the maintainers before opening a
public issue. Do not include sensitive diagram source, credentials, or private
image URLs in a report.

The GitHub Pages application performs conversion locally and does not upload
diagram source. Any future hosted API will document retention, rate limits, and
remote-asset restrictions before public launch.

If confidential content is accidentally published in an example, release, or
repository history, do not open a public issue. Report it privately so the
maintainers can remove public references, rewrite history, purge hosted
artifacts, and coordinate cache removal where necessary.
